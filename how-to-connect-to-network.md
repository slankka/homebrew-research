## How to connect to network
Some contents are generated by AI. And the another contents are written by me.

> 根据我对 libcurl 在 Nintendo Switch 上的移植代码分析，我发现了 nro 模式和 sysmodule 模式下网络访问差异的关键问题：

## 问题分析

> 在 curl-7.69.1/switch-curl.patch 和相关文件中，我发现了以下关键信息：

### 1. 网络初始化的差异

**nro 模式**：
- Homebrew 应用（.nro 文件）运行时，系统会自动处理基础的网络服务初始化
- 应用可以直接使用网络功能，无需手动初始化底层服务

**sysmodule 模式**：
- Sysmodule 作为后台进程运行，需要手动初始化所需的系统服务
- 缺少自动的网络服务初始化，需要显式调用相关初始化函数

### 2. libcurl 中的网络初始化

在 /curl-7.69.1/lib/vtls/libnx.c libnx.c:820 函数：

```c
static int Curl_libnx_init(void)
{
  Result rc = 0;
  rc = sslInitialize(0x3);
  if(R_SUCCEEDED(rc))
    rc = csrngInitialize();
  return R_SUCCEEDED(rc);
}
```

这个函数只初始化了 SSL 和随机数生成器服务，但**没有初始化基础的网络服务**。

### 3. 代理配置获取

在/curl-7.69.1/lib/hos.c Curl_os_get_system_proxy 函数确实调用了 `nifmInitialize`，但这只是为了获取代理配置，而不是为了基础网络功能。

## 解决方案

**是的，sysmodule 模式需要手动执行网络初始化代码。** 你需要在 sysmodule 的初始化代码中添加以下调用：

```c
// 初始化基础网络服务
socketInitializeDefault();

// 如果需要网络配置信息，也可以初始化 nifm
nifmInitialize(NifmServiceType_System); // 注意：sysmodule 使用 System 类型
```

在 sysmodule 退出时，也需要相应的清理：

```c
nifmExit();
socketExit();
```

### 关键差异总结

1. **nro 模式**：系统自动处理网络服务初始化
2. **sysmodule 模式**：需要手动调用 `socketInitializeDefault()` 等函数
3. **libcurl 的 nifm 调用**：仅用于获取代理配置，不负责基础网络初始化
4. **服务类型**：sysmodule 应使用 `NifmServiceType_System` 而不是 `NifmServiceType_User`

这解释了为什么相同的 libcurl 代码在 nro 模式下能正常工作，但在 sysmodule 模式下无法访问网络。

bool nifmInternetIsConnected() {
    Result rs = nifmInitialize(NifmServiceType_User);
    if (R_FAILED(rs)) {
        printf("Failed to get Internet connection status. Error code %x\n", rs);
        return false;
    }

    NifmInternetConnectionStatus nifmICS;
    rs = nifmGetInternetConnectionStatus(NULL, NULL, &nifmICS);
    nifmExit();

    if (R_FAILED(rs)) {
        printf("nifmGetInternetConnectionStatus failed with error code %x\n", rs);
    }

    if (nifmICS != NifmInternetConnectionStatus_Connected) {
        return false;
    }

    return true;
}
